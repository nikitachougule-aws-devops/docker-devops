## Dockerfile

It is a text file that contains instructions to build a Docker image.

It defines:

- Base image  
- Application code  
- Dependencies  
- Commands to run  

---

## 🔹 Basic Dockerfile Structure

```dockerfile
# Use base image
FROM ubuntu:latest

# Install dependencies
RUN apt-get update && apt-get install -y nginx

# Copy files
COPY . /app

# Set working directory
WORKDIR /app

# Run command
CMD ["echo", "Hello Docker"]
```
---

## Common Dockerfile Instructions

### 1️⃣ FROM

Defines base image
```bash
FROM node:18
```

### 2️⃣ RUN

Executes commands during build
```bash
RUN apt-get update
```

### 3️⃣ COPY

Copies files from local to container
```bash
COPY . /app
```

### 4️⃣ WORKDIR

Sets working directory
```bash
WORKDIR /app
```

### 5️⃣ CMD

Default command when container runs
```bash
CMD ["node", "app.js"]
```

### 6️⃣ ENTRYPOINT (Optional)

Defines main command
```bash
ENTRYPOINT ["python3"]
```
---

## Build Docker Image
```bash
docker build -t myapp .
```
- -t → tag name
- . → current directory

---

## Run Container
```bash
docker run myapp
```
---

## Real-Time Example (Node.js App)
### Dockerfile:
```bash
FROM node:18

WORKDIR /app

COPY package.json .

RUN npm install

COPY . .

CMD ["node", "app.js"]
```
---

## Dockerfile Workflow
```bash
Dockerfile → docker build → Image → docker run → Container
```
---
## Best Practices
- Use lightweight images (e.g., alpine)
- Minimize number of layers
- Use .dockerignore
- Avoid running as root
- Use multi-stage builds

## Summary
- Dockerfile is used to build images
- Contains step-by-step instructions
- Helps automate container creation
- Essential for DevOps workflows
